# ✅ Checklist d'intégration au repo partagé

- [ ] Accès collaborateur sur `MaatougAyoub/AgriFlow` confirmé
- [ ] Repo partagé cloné localement
- [ ] Branche `feature/module5-collaborations` créée
- [ ] Structure du repo partagé comprise (mono-repo ou multi-modules)

## Adaptation du code

- [ ] Packages renommés si nécessaire (ex: `entities` → `tn.esprit.agriflow.module5.entities`)
- [ ] `pom.xml` adapté à la structure du repo partagé (parent POM si existant)
- [ ] Imports mis à jour (si changement de packages)
- [ ] Configuration DB harmonisée avec les autres modules
- [ ] Dépendances communes utilisées (versions alignées)

## Base de données

- [ ] Script SQL `module5-schema.sql` testé
- [ ] Foreign keys vers `users` validées (existe dans Module 1)
- [ ] Pas de conflit de noms de tables avec autres modules

## Documentation

- [ ] README du module créé/mis à jour
- [ ] CONTRIBUTING.md rempli
- [ ] Commentaires de code en français (si convention équipe)
- [ ] Diagrammes UML à jour (si demandés)

## Tests et validation

- [ ] Compilation réussie : `mvn clean compile`
- [ ] Application console se lance : `java -cp target/classes tests.Main`
- [ ] CRUD complet testé manuellement
- [ ] Connexion MySQL OK (port 3306, base `agriflow`)
- [ ] Pas de warnings Maven critiques

## Git et GitHub

- [ ] Branche feature créée localement
- [ ] Commits atomiques et messages clairs
- [ ] Branche pushée vers GitHub : `git push -u origin feature/module5-collaborations`
- [ ] Branche visible sur GitHub (vérifier dans Branches)
- [ ] Pas de fichiers inutiles committé (`.class`, `target/`, `.idea/`)

## Pull Request

- [ ] PR créé sur `MaatougAyoub/AgriFlow`
- [ ] Titre descriptif : `feat(module5): Add Collaborations module`
- [ ] Description complète (fonctionnalités, structure, user stories)
- [ ] Reviewers assignés (MaatougAyoub + autres membres)

- [ ] Lié au milestone "Sprint 0" (si existant)

## Communication équipe

- [ ] Informer l'équipe de la PR (Discord/WhatsApp/Email)
- [ ] Expliquer les dépendances vers Module 1 (table `users`)

## Après validation

- [ ] Merger le PR (ou attendre validation chef d'équipe)
